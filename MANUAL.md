# 업무망 점검기 사용 설명서

학교 **교사 업무망**에 무단으로 연결된 장비(공유기·무선 AP·전화기)나 다른 망과의 **혼선**을
자동으로 찾아 알려 주는 데스크톱 앱입니다. 네트워크를 잘 몰라도 **버튼 한 번**이면 됩니다.

- 결과 해석이 궁금하면 → [4. 결과 읽는 법](#4-결과-읽는-법)
- 상시 감시를 걸고 싶으면 → [6. 감시 모드](#6-감시-모드)

## 1. 소개

학교 망은 **교사 업무망 / 학생 유선망 / 학생 무선망 / 전화망** 네 개로 나뉘어 서로 **분리**되어야 합니다.
특히 **교사 업무망에는 무선(공유기·AP)을 연결하면 안 됩니다.**

이 앱은 업무망에 연결된 PC에서 실행하면, 같은 망에 있는 장비들을 훑어:

- 🔴 전화업자가 꽂은 **공유기**, 교사가 설치한 **무선 AP** 같은 **비인가 장비**
- 🟡 다른 망(특히 학생 무선망·전화망)이 잘못 연결된 **혼선**

을 찾아 **빨간색으로 표시하고, 상세 보고**해 줍니다. 설치가 필요 없고 `업무망점검기.exe` 하나만 더블클릭하면 됩니다.

---

## 2. 빠른 시작

1. `업무망점검기.exe`를 **더블클릭**합니다.
2. **[🔍 내 네트워크 스캔]** 버튼을 누릅니다.
   - 지금 이 PC가 연결된 네트워크를 자동으로 점검합니다. (주소 입력 필요 없음)
3. 잠시 기다리면 결과 표가 채워집니다.
   - **빨간 줄**이 있으면 = 이상 장비. 아래 **[⚠ 상세 보고]** 버튼으로 자세히 봅니다.
   - 빨간 줄이 없으면 = `✅ 이상 없음`.
4. 필요하면 **[CSV 저장]**으로 보고서를 내려받습니다.

> 💡 반드시 **점검하려는 업무망에 연결된 PC**에서 실행하세요. (정보부장 PC 권장)

## 3. 화면 구성

- **[🔍 내 네트워크 스캔]** — 현재 연결된 네트워크를 점검 시작
- **[중지]** — 진행 중인 스캔 중단
- **[⚠ 상세 보고]** — 이상 항목의 상세 내용(원칙·근거·조치) 보기
- **[CSV 저장]** — 결과를 엑셀로 열 수 있는 파일로 저장
- **감시 모드 / 주기(분) / 윈도우 시작 시 자동 실행** — [6. 감시 모드](#6-감시-모드) 참고
- 아래쪽 **상태줄** — 진행 상황과 `이상 N건 (혼선 X · 비인가 Y)` 요약 표시

결과 표의 열:

| 열 | 뜻 |
|----|----|
| IP | 장비 주소 |
| MAC | 장비 고유번호 |
| 종류 | 공유기(의심)/무선AP(의심)/인터넷전화기/미상 |
| 신뢰도% | 그 판정의 확신 정도(높을수록 확실) |
| 제조사 | MAC으로 추정한 제조사 |
| 포트·근거 | 판정 근거 |

## 4. 결과 읽는 법

**빨간 줄 = 주목해야 할 장비**입니다. **[⚠ 상세 보고]**를 누르면 이상을 두 가지로 구분해 알려 줍니다.

### ① 비인가 장비 연결
업무망에 무단으로 꽂힌 **공유기·무선 AP**입니다.
- 예: `[심각] 비인가 무선/공유기 장비 — IP 10.20.30.45 · ipTIME`
- 조치: 그 IP의 스위치 포트에서 **장비를 분리**하고 배선을 확인합니다.

### ② 혼선 (망 간 연결)
분리돼야 할 다른 망이 업무망에 닿은 경우입니다.
- **업무망에서 DHCP 감지** → DHCP를 쓰는 **학생 무선망이 유입**됐을 가능성
- **전화기 발견** → **전화망이 업무망과 연결**됐을 가능성
- 조치: 해당 IP의 연결 경로를 추적해 망 분리를 확인합니다.

> **신뢰도**가 90% 이상이면 거의 확실, 50~70%면 추가 확인이 필요한 의심입니다.

## 5. 보고서 저장(CSV)

**[CSV 저장]**을 누르면 `업무망점검_날짜.csv` 파일이 만들어집니다.

- 엑셀에서 바로 열립니다(한글 깨짐 없음).
- 맨 위에 **이상 요약과 위반 상세표**(심각도·유형·IP·MAC·조치)가 들어가, 교육청 보고·점검 대장에 그대로 쓸 수 있습니다.

---

## 6. 감시 모드

정보부장 PC에 두고 **상시 감시**할 때 사용합니다.

1. **감시 모드**를 체크하고 **주기(분)**를 정합니다. (예: 10분)
2. **윈도우 시작 시 자동 실행(감시)**를 체크하면, 로그인할 때마다 자동으로 켜져 트레이(시계 옆)에서 조용히 감시합니다.
3. 승인되지 않은 **새 이상 장비**가 나타나면 **트레이 알림 + 경고음**으로 즉시 알려 줍니다.
   - 알림에 `유형: 망 혼선 의심 / 비인가 장비 연결`이 표시됩니다.

창을 최소화하면 트레이로 들어갑니다. 트레이 아이콘을 더블클릭하면 다시 열립니다.

### 정상 장비를 승인하기
정상인데 의심으로 잡힌 장비는, 표에서 그 줄을 **우클릭 → "이 장비를 정상(승인)으로 등록"**하면 다음부터 경고하지 않습니다.

## 7. 무엇을 점검하나 — 4개 망 분리 원칙

| 망 | 특징 | 업무망에서 보이면 |
|----|------|------------------|
| 교사 업무망 | 유선·고정 IP·무선/ DHCP 없음 | (정상) |
| 학생 유선망 | 유선·고정 IP | 혼선 의심 |
| 학생 무선망 | 무선·**DHCP 사용** | DHCP 감지 = 혼선 |
| 전화망 | 유선(전화기) | 전화기 발견 = 혼선 |

업무망은 원래 무선도 DHCP도 없어야 하므로, 그런 신호가 잡히면 곧 이상입니다.

---

## 8. 고급 옵션(대역 직접 지정)

> 네트워크를 아는 사용자만 사용하세요. 일반 교사는 필요 없습니다.

다른 대역(예: 학생망)을 직접 점검하려면 **Ctrl + Shift + A**를 누르면 숨겨진 **대상 대역** 입력칸이 나타납니다.
`10.20.30.0/24` 형식으로 입력하면 그 대역을 스캔합니다. (비우면 다시 자동 탐지)

---

## 9. 권한과 주의사항

- 일반 스캔(공유기·AP·전화기 탐지)은 **보통 권한**으로 동작합니다.
- **DHCP 정밀 탐지**는 Windows DHCP 기능과 충돌할 수 있어, **관리자 권한으로 실행**하면 정확도가 올라갑니다.
- ⚠️ **본인이 관리하거나 점검 권한이 있는 학교 망에서만** 사용하세요.

## 10. 자주 묻는 질문

**Q. 교무실에 저가 허브로 포트를 늘렸는데 오탐되나요?**
A. 아니요. 더미 허브는 망을 새로 만들지 않아(투명) 공유기로 오인하지 않습니다. 허브 뒤 장비도 정상적으로 점검됩니다.

**Q. 정상 장비가 의심으로 떴어요.**
A. 그 줄을 우클릭해 **정상(승인)**으로 등록하면 다음부터 제외됩니다.

**Q. 아무것도 안 잡혀요.**
A. ① 업무망에 연결된 PC인지 ② 방화벽이 스캔을 막지 않는지 확인하세요. 그래도 안 되면 관리자 권한으로 실행해 보세요.

**Q. 신뢰도가 낮게(50~70%) 나왔어요.**
A. 한 가지 신호만 잡힌 경우입니다. 의심은 되지만 추가 확인이 필요하다는 뜻이니, 상세 보고의 근거를 보고 현장 확인하세요.

**Q. 학생 무선망과 연결됐는지도 알 수 있나요?**
A. 네. 업무망에서 **DHCP가 감지되면** 혼선으로 보고합니다. 다른 대역을 직접 확인하려면 고급 옵션(8장)을 쓰세요.
